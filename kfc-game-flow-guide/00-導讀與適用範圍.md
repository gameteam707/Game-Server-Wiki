# KFC 遊戲流程指南 — 導讀與適用範圍

## 這份文件的目的

本系列文件是 KFC 遊戲平台的**遊戲流程架構指南**，涵蓋：

1. **通用遊戲流程架構** — 所有遊戲類型共用的生命週期、服務交互、狀態機模型
2. **單人遊戲流程（Spin Lifecycle）** — 現有 CashMachine / XiaoMali 的 V2 Spin 完整流程
3. **Arena 賽事遊戲** — 多人對戰投注遊戲，GameService 作為中間人角色的完整設計

## 適用對象

- 準備開發 GameService / GameAPI Arena 整合功能的工程師
- 需要理解下注 / 結算流程的前後端開發者
- 進行 Code Review 時需要對照架構設計的團隊成員

## 兩種遊戲模式對比

| 維度 | 單人 PvE（Spin） | Arena 賽事投注 |
|------|-----------------|---------------|
| 代表遊戲 | CashMachine、XiaoMali | Battle Arena |
| 結果來源 | RNG（math-lib） | 外部真實賽事信號（由 arena-service 管理） |
| 回合歸屬 | 每個玩家獨立一局 | 多位玩家共享同一場賽事 |
| 賠率模型 | 固定 RTP（例：95%） | 動態賠率（RTP-based 初始 + 投注量調整） |
| 下注觸發 | 玩家自行按下 Spin | 管理員開盤，玩家在時間窗口內下注 |
| 結算方式 | 即時（Spin 完成即結算） | 延遲（賽事結束後由 arena-service 通知結算） |
| GameService 角色 | 核心邏輯主體（RNG + Wallet） | 中間人（Wallet 操作 + 推播轉發） |
| 賠率計算 | GameService 內部（math-lib） | arena-service 負責（Redis Lua Script） |
| 回合管理 | 無（每次 Spin 即一局） | arena-service + Admin Backend |

## 閱讀方式建議

1. **搭配程式碼一起看**：文件中會標注對應的檔案路徑，建議開 IDE 對照
2. **先讀通用流程再讀特定遊戲**：01 是所有遊戲的共同基礎
