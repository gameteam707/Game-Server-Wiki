# KFC 遊戲流程通用指南 — 導讀與適用範圍

## 這份文件的目的

本系列文件是 KFC 遊戲平台的**遊戲流程架構指南**，涵蓋：

1. **通用遊戲流程架構**——所有遊戲類型共用的生命週期、服務交互、狀態機模型
2. **單人遊戲流程（Spin Lifecycle）**——現有 CashMachine / XiaoMali 的 V2 Spin 完整流程
3. **多人遊戲流程（Round Lifecycle）**——多人共享回合的投注型遊戲流程
4. **功夫雞（Kung Fu Chicken）設計規格**——第一款多人共享回合遊戲的完整設計

## 適用對象

- 準備開發新遊戲功能的工程師
- 需要理解下注/結算流程的前後端開發者
- 進行 Code Review 時需要對照架構設計的團隊成員

## 兩種遊戲模式對比

| 維度 | 單人 PvE（Spin） | 多人共享回合（Round） |
|------|-----------------|---------------------|
| 代表遊戲 | CashMachine、XiaoMali | Kung Fu Chicken（功夫雞） |
| 結果來源 | RNG（math-lib） | 外部真實賽事信號 |
| 回合歸屬 | 每個玩家獨立一局 | 多位玩家共享同一回合 |
| 賠率模型 | 固定 RTP（例：95%） | 動態 Parimutuel（依投注池分佈） |
| 下注觸發 | 玩家自行按下 Spin | 管理員開盤，玩家在時間窗口內下注 |
| 結算方式 | 即時（Spin 完成即結算） | 批次（回合結束後一次性結算所有下注） |
| 狀態機 | BetRecord 個人狀態 | Round 全局狀態 + 個人 Bet 狀態 |

## 閱讀方式建議

1. **搭配程式碼一起看**：文件中會標注對應的檔案路徑，建議開 IDE 對照
2. **先讀通用流程再讀特定遊戲**：01 是所有遊戲的共同基礎
3. **Sequence Diagram 是核心**：每個流程都附有 Mermaid 圖，這是理解服務交互最快的方式
