# 06 — 功夫雞：實作計畫

## 實作階段總覽

```
Phase 1: Domain Layer          ← 純業務邏輯，零外部依賴
    ↓
Phase 2: UseCase Layer         ← 業務流程編排，定義 Ports
    ↓
Phase 3: Adapter Layer         ← gRPC Server + Repo + Redis + Proto
    ↓
Phase 4: Connector 整合        ← WebSocket Handler + Routes
    ↓
Phase 5: Admin Backend 整合    ← 回合管理 API
    ↓
Phase 6: Push 推播整合          ← 賠率 + 狀態廣播
```

### UseCase 2：`kfc_place_bet`（下注）

```
gameservice/internal/usecase/kfc_place_bet/
├── contract.go
├── uc.go
└── ports/
    ├── bet_repository.go
    └── round_reader.go
```

**Contract 設計**：

```go
type UseCase interface {
    Execute(ctx context.Context, cmd *Command) (*Result, error)
}

type Command struct {
    RoundID     string
    UserID      string
    GameID      string
    Option      entity.BetOption
    BetAmount   float64
    Currency    string
    IID         string
    PlayerToken string
    IsBot       bool
}

type Result struct {
    BetID       string
    RoundID     string
    Option      entity.BetOption
    BetAmount   float64
    Odds        float64      // 下注時即時賠率
    Balance     float64      // 扣款後餘額
    CurrentOdds *entity.RoundOdds
}
```

**核心邏輯（uc.go）**：

```
1. 驗證 Round status == Open
2. 驗證 BetOption 合法
3. 取得 Integrator endpoints (ConfigCache)
4. Wallet Debit (WalletClient)
5. Redis HINCRBY 更新投注池（原子）
6. Double Check Round status（防止 Debit 期間封盤）
   └── 若已 Closed → Cancel Debit + Redis HINCRBY 反向扣減
7. MongoDB 寫入 KFCBet (status=Confirmed)
8. 計算即時賠率
9. 非同步：Kafka 發布 odds update + bet-records
10. 回傳結果
```

### UseCase 3：`kfc_settle`（結算）

```
gameservice/internal/usecase/kfc_settle/
├── contract.go
├── uc.go
└── ports/
    └── bet_settler.go
```

**Contract 設計**：

```go
type UseCase interface {
    Execute(ctx context.Context, cmd *Command) (*Result, error)
}

type Command struct {
    RoundID string
    Result  entity.BetOption  // "wala" / "melo" / "draw"
}

type Result struct {
    Round        *entity.Round
    WinnersCount int
    LosersCount  int
    TotalPayout  float64
    FinalOdds    *entity.RoundOdds
}
```

**核心邏輯（uc.go）**：

```
1. 驗證 Round status == Closed
2. 從 Redis 讀取最終投注池
3. 計算最終賠率（Parimutuel 公式）
4. 更新 Round: status=Settled, result
5. 查詢所有 Bets
6. 遍歷所有 Bet：
   ├── 贏家：winAmount = betAmount × finalOdds
   │   → Wallet Credit → Update Bet status=Distributed
   │   └── Credit 失敗 → Outbox
   └── 輸家：winAmount = 0
       → Update Bet status=Settled
7. 所有 Bet 處理完成 → Round status=Completed
8. 非同步：Kafka 發布 round.settled + bet.result events
9. 清理 Redis 投注池
```

### UseCase 4：`kfc_odds`（賠率計算 — 純函式）

```
gameservice/internal/usecase/kfc_odds/
├── contract.go
└── odds.go
```

```go
// odds.go — 純函式，無外部依賴
func Calculate(pool *entity.BetPool, houseEdge float64) *entity.RoundOdds {
    total := pool.Wala + pool.Melo + pool.Draw
    if total == 0 {
        return &entity.RoundOdds{} // 全零
    }
    effective := total * (1 - houseEdge)
    return &entity.RoundOdds{
        WalaOdds:  safeDiv(effective, pool.Wala),
        MeloOdds:  safeDiv(effective, pool.Melo),
        DrawOdds:  safeDiv(effective, pool.Draw),
        TotalPool: total,
        WalaPool:  pool.Wala,
        MeloPool:  pool.Melo,
        DrawPool:  pool.Draw,
    }
}

func CalculatePayout(betAmount, odds float64) float64 {
    return betAmount * odds
}

func safeDiv(numerator, denominator float64) float64 {
    if denominator == 0 {
        return 0
    }
    return numerator / denominator
}
```

## Phase 6：Push 推播整合

### 賠率更新推播

```
PlaceBet UseCase (postBetAsync goroutine)
    → Kafka "kfc-odds-updates" topic
    → PushProducer Consumer
    → PushProducer.Broadcast(route="kfc.odds.update", payload)
    → Kafka "ws-event-push" topic
    → Connector Kafka Consumer
    → WebSocket Broadcast to all connected clients
```

### 回合狀態推播

```
RoundMgmt UseCase (Open/Close/Settle/Cancel)
    → Kafka "kfc-round-events" topic
    → 同上路徑 → Connector → WebSocket Broadcast
```

### 個人結算結果推播

```
Settle UseCase (批次結算完成後)
    → Kafka "kfc-bet-results" topic
    → PushProducer Consumer
    → PushProducer.SendToUser(userID, route="kfc.bet.result", payload)
    → 同上 → Connector → 僅推送給該玩家
```

## 測試策略

### UseCase 單元測試（優先級 P0）

| UseCase | 測試重點 |
|---------|---------|
| `kfc_odds` | 邊界條件：全零池、單一選項有人投注、極端分佈 |
| `kfc_place_bet` | 正常下注、Round 非 Open 拒絕、Debit 失敗回滾、Double Check 退款 |
| `kfc_settle` | 正常結算、Credit 失敗走 Outbox、全部輸家（winAmount=0） |
| `kfc_round_mgmt` | 狀態轉換合法性、Cancel 全額退款 |

### 整合測試（優先級 P1）

| 測試場景 | 涵蓋範圍 |
|---------|---------|
| Round 完整生命週期 | Create → Open → PlaceBet × N → Close → Settle → Completed |
| 回合取消退款 | Open → PlaceBet × N → Cancel → 所有 Bet Refunded |
| 併發下注 | 多 goroutine 同時 PlaceBet → 投注池一致性 |
| 結算補償 | Settle 中途 Credit 失敗 → Outbox → 重試成功 |

### E2E 測試（優先級 P2）

使用 `Special_Game_Pipeline` docker-compose 環境：
1. dummyPlatform 提供 Wallet API
2. Connector → GameService → Wallet 全鏈路
3. Admin Backend → GameService gRPC 回合管理
