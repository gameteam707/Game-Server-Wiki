# 12 — 可觀測性（Observability）

## 什麼是可觀測性？

可觀測性 = **能從系統的外部輸出，推斷系統的內部狀態**。

當生產環境出問題時，你不能 SSH 進去加 `fmt.Println` 除錯。你需要三種工具：

```
可觀測性的三大支柱
├── Logs（日誌）    → 發生了什麼事？
├── Metrics（指標） → 系統目前的數字是多少？
└── Traces（追蹤）  → 一個請求經過了哪些服務、花了多少時間？
```

```
位置：/go-observability/
```

### Tracing（分散式追蹤）

Trace 記錄一個請求從進入系統到完成的 **完整路徑**。

```
一次玩家下注的 Trace：

Trace ID: abc-123
├── [gameapi] POST /api/v1/spin                    120ms
│   ├── [gameapi] Parse Request                      2ms
│   ├── [gameapi] gRPC call to gameservice          95ms
│   │   ├── [gameservice] Spin UseCase              80ms
│   │   │   ├── [gameservice] MongoDB: find session  5ms
│   │   │   ├── [gameservice] HTTP: debit wallet    30ms
│   │   │   ├── [gameservice] Math: calculate spin  10ms
│   │   │   ├── [gameservice] HTTP: credit wallet   25ms
│   │   │   └── [gameservice] MongoDB: save record   8ms
│   │   └── [gameservice] Serialize response         2ms
│   └── [gameapi] Format HTTP response               3ms
```

每一段叫做一個 **Span**。Span 之間有父子關係，串起來就是完整的 Trace。

### OTel 在 Go 中的使用

```go
// go-observability 提供的初始化函式
func InitTracer(serviceName string, exporterURL string) {
    exporter := otlptrace.New(...)
    tp := trace.NewTracerProvider(
        trace.WithBatcher(exporter),
        trace.WithResource(resource.NewWithAttributes(
            semconv.ServiceNameKey.String(serviceName),
        )),
    )
    otel.SetTracerProvider(tp)
}
```

```go
// 在 gameservice main.go 中初始化
func main() {
    observability.InitTracer("gameservice", cfg.OTelExporterURL)
    // ... 啟動服務
}
```

### Middleware 自動追蹤

go-observability 提供了 Gin 和 gRPC 的 middleware，自動為每個請求建立 Span：

```go
// Gin middleware：自動追蹤每個 HTTP 請求
router.Use(otelgin.Middleware("gameapi"))

// gRPC interceptor：自動追蹤每個 gRPC 呼叫
grpc.NewServer(
    grpc.UnaryInterceptor(otelgrpc.UnaryServerInterceptor()),
)
```

## 2. Metrics（指標）

### Prometheus 指標

```go
// 檔案：gameservice/internal/infrastructure/metrics/init.go

var (
    SpinTotal = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name: "game_spin_total",
            Help: "Total number of spins",
        },
        []string{"game_code", "status"},  // 標籤
    )

    SpinDuration = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "game_spin_duration_seconds",
            Help:    "Spin request duration in seconds",
            Buckets: prometheus.DefBuckets,
        },
        []string{"game_code"},
    )
)

func Init() {
    prometheus.MustRegister(SpinTotal, SpinDuration)
}
```

### 在業務程式碼中記錄指標

```go
func (uc *spinUseCase) Execute(ctx context.Context, cmd *Command) (*Result, error) {
    start := time.Now()
    defer func() {
        metrics.SpinDuration.WithLabelValues(cmd.GameCode).Observe(time.Since(start).Seconds())
    }()

    result, err := uc.doSpin(ctx, cmd)
    if err != nil {
        metrics.SpinTotal.WithLabelValues(cmd.GameCode, "error").Inc()
        return nil, err
    }

    metrics.SpinTotal.WithLabelValues(cmd.GameCode, "success").Inc()
    return result, nil
}
```

### 常見指標類型

| 類型 | 用途 | 範例 |
|------|------|------|
| **Counter** | 只會增加的計數器 | 請求總數、錯誤數 |
| **Gauge** | 可增可減的數值 | 當前連線數、佇列長度 |
| **Histogram** | 數值分佈 | 請求延遲、回應大小 |

## 3. Logging（日誌）

本專案使用 Go 內建的 `slog`（structured logging）：

```go
import "log/slog"

// 結構化日誌：每個訊息帶有可搜尋的 key-value
slog.InfoContext(ctx, "spin completed",
    "sessionId", cmd.SessionID,
    "gameCode", cmd.GameCode,
    "betAmount", cmd.BetAmount,
    "winAmount", result.WinAmount,
)

// 輸出：
// {"time":"2024-03-20T14:22:00Z","level":"INFO","msg":"spin completed",
//  "sessionId":"sess_123","gameCode":"slot-fortune","betAmount":10,"winAmount":50}
```

**為什麼用結構化日誌而非 `fmt.Printf`？**

```
// ❌ 純文字日誌（難以搜尋和分析）
fmt.Printf("Spin completed for session %s, bet %f, win %f\n", ...)

// ✅ 結構化日誌（可以用 Loki/ELK 搜尋 sessionId=sess_123）
slog.Info("spin completed", "sessionId", "sess_123", ...)
```
