# KFC Project — Junior Developer 導讀指南

## 這份文件是給誰看的？

如果你是剛加入團隊的 Junior Developer，對 Go 有基礎認識但還沒接觸過大型專案架構，這份文件就是為你準備的。

## 為什麼需要這份文件？

這個專案由 Senior Developer 規劃，採用了業界成熟的架構模式（Clean Architecture、Hexagonal Architecture、DDD 概念等）。初次接觸可能會覺得「為什麼要分這麼多層？」「為什麼不能直接寫？」——這份文件會帶你理解每個設計決策背後的 **為什麼**。

## 建議閱讀順序

```
第一階段：建立全局觀
├── 01-專案總覽.md          ← 先知道這個系統在做什麼
├── 02-Clean-Architecture.md ← 理解核心架構哲學
└── 03-目錄結構與分層.md     ← 對照程式碼看每一層在哪裡

第二階段：深入每一層（由內而外）
├── 04-Domain-Layer.md       ← 最核心、最穩定的一層
├── 05-UseCase-Layer.md      ← 業務邏輯怎麼組織
├── 06-Adapter-Layer.md      ← 如何與外部世界溝通
└── 07-Infrastructure-Layer.md ← 框架、資料庫、設定

第三階段：橫切關注點
├── 08-依賴注入.md           ← 怎麼把各層組裝起來
├── 09-API設計與Middleware.md ← HTTP/gRPC 怎麼設計
├── 10-資料庫與Repository.md  ← 資料怎麼存取
└── 11-微服務通訊.md          ← 服務之間怎麼對話

第四階段：營運面
├── 12-可觀測性.md           ← 怎麼知道系統是否健康
├── 13-部署與容器化.md        ← 怎麼上線
└── 14-測試策略.md           ← 怎麼確保品質

第五階段：你的下一步
└── 15-學習路線圖.md         ← 技能補齊建議與學習資源
```

## 閱讀方式建議

1. **搭配程式碼一起看**：每份文件都會標注對應的檔案路徑，建議用 IDE 打開專案邊看邊對照
2. **不要死背**：理解「為什麼」比記住「怎麼做」更重要
3. **動手改改看**：理解一個模式最好的方式是試著自己加一個小功能
4. **有問題就問**：這份文件無法涵蓋所有細節，遇到不懂的地方記下來跟團隊討論
