# 08 — 依賴注入（Dependency Injection）

## 什麼是依賴注入？

簡單來說：**不要自己 new 依賴，讓外面傳進來。**

```go
// ❌ 自己建立依賴（緊耦合）
type GameUseCase struct {
    repo *MongoGameRepository   // 直接依賴具體實作
}

func NewGameUseCase() *GameUseCase {
    client, _ := mongo.Connect(...)              // UseCase 不該知道怎麼連 DB
    repo := NewMongoGameRepository(client)       // UseCase 不該知道用 MongoDB
    return &GameUseCase{repo: repo}
}
```

```go
// ✅ 依賴注入（鬆耦合）
type GameUseCase struct {
    repo ports.GameRepository   // 依賴介面
}

func NewGameUseCase(repo ports.GameRepository) *GameUseCase {
    return &GameUseCase{repo: repo}   // 從外部注入
}
```

## 本專案的 DI 實作：Container Pattern

本專案不使用 DI 框架（如 Wire、Dig），而是用最直覺的方式——**一個大的 Container struct**。

```go
// 檔案：admin-backend/server/internal/infrastructure/di/container.go

type Container struct {
    // Handlers（給 Router 使用）
    AuthHandler        *http.AuthHandler
    GameHandler        *http.GameHandler
    CurrencyHandler    *http.CurrencyHandler
    IntegratorHandler  *http.IntegratorHandler
    // ... 更多 Handlers

    // UseCases（給 Handler 和 Middleware 使用）
    AuditLogCreateUC   audit.CreateUseCase

    // Repositories（給 Middleware 使用）
    PermGroupRepo      ports.PermissionGroupRepository

    // Config
    JWTSecret          string
}
```

### 組裝過程（由內而外）

```go
func NewContainer(db *mongo.Database, redis *redis.Client, cfg *config.Config) *Container {
    // ====== 第一步：建立 Adapter 層（Repository、Gateway）======
    gameRepo := mongoRepo.NewGameRepository(db)
    userRepo := mongoRepo.NewAdminUserRepository(db)
    currencyRepo := mongoRepo.NewCurrencyRepository(db)
    permGroupRepo := mongoRepo.NewPermissionGroupRepository(db)
    auditLogRepo := mongoRepo.NewAuditLogRepository(db)
    loginLimiter := provider.NewRedisLoginLimiter(redis)
    gameAPIClient := gateway.NewGameAPIClient(cfg.GameAPIURL)

    // ====== 第二步：建立 UseCase 層 ======
    // 把 Repository（介面）注入到 UseCase
    loginUC := auth.NewLoginUseCase(userRepo, loginLimiter, cfg.JWTSecret, cfg.JWTExpiry)
    meUC := auth.NewMeUseCase(userRepo)
    gameCreateUC := game_mgmt.NewCreateUseCase(gameRepo)
    gameListUC := game_mgmt.NewListUseCase(gameRepo)
    currencyCreateUC := currency_mgmt.NewCreateUseCase(currencyRepo)
    auditLogCreateUC := audit.NewCreateUseCase(auditLogRepo)

    // ====== 第三步：建立 Adapter 層（Handler）======
    // 把 UseCase（介面）注入到 Handler
    authHandler := http.NewAuthHandler(loginUC, meUC)
    gameHandler := http.NewGameHandler(gameCreateUC, gameListUC)
    currencyHandler := http.NewCurrencyHandler(currencyCreateUC)

    // ====== 第四步：組裝 Container ======
    return &Container{
        AuthHandler:      authHandler,
        GameHandler:      gameHandler,
        CurrencyHandler:  currencyHandler,
        AuditLogCreateUC: auditLogCreateUC,
        PermGroupRepo:    permGroupRepo,
        JWTSecret:        cfg.JWTSecret,
    }
}
```

### 組裝流程圖

```
mongo.Database ──┐
                 ├──▶ GameRepository ──┐
redis.Client ───┘                      │
                                       ├──▶ CreateUseCase ──┐
config.Config ─────────────────────────┘                     │
                                                             ├──▶ GameHandler
                                       ┌──▶ ListUseCase ────┘
mongo.Database ──▶ GameRepository ─────┘

最終：Container { GameHandler, ... } ──▶ Router
```

## 為什麼不用 DI 框架？

Go 社群有 Google Wire（編譯時 DI）和 Uber Dig（執行時 DI）等框架，但本專案選擇手動 DI：

| 手動 DI | DI 框架 |
|---------|---------|
| 程式碼明確，一目了然 | 需要學習框架 API |
| 編譯時就能檢查錯誤 | 可能有執行時錯誤（Dig） |
| 沒有額外依賴 | 增加專案依賴 |
| 新增元件要改 Container | 自動解析依賴（Wire） |

對於中小型專案，手動 DI 的可讀性優勢通常大於自動化的便利性。

## Seed 初始資料

Container 建立時還有一個重要任務——確保系統有初始資料（如超級管理員帳號）：

```go
func NewContainer(...) *Container {
    // ... 建立所有依賴 ...

    // Seed: 確保超級管理員存在
    seedSuperAdmin(ctx, userRepo, cfg.SuperAdminUser, cfg.SuperAdminPass)

    return &Container{...}
}

func seedSuperAdmin(ctx context.Context, repo ports.AdminUserRepository, username, password string) {
    existing, _ := repo.FindByUsername(ctx, username)
    if existing != nil {
        return  // 已存在，跳過
    }

    hashed, _ := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    user := &entity.AdminUser{
        ID:             generateID(),
        Username:       username,
        HashedPassword: string(hashed),
        Role:           entity.RoleSuperAdmin,
    }
    repo.Create(ctx, user)
}
```

## 給 Junior 的學習重點

### 你需要理解的概念

1. **依賴注入 vs 依賴查找**：注入是「別人給我」，查找是「我自己去找」
2. **控制反轉 (IoC)**：元件不再自己控制依賴的建立，控制權交給外部
3. **組裝點 (Composition Root)**：整個應用程式只有一個地方知道所有實作的具體型別——就是 DI Container

### 為什麼要這樣做？

想像你要測試 `GameHandler`：

```go
// 因為 Handler 依賴的是介面，你可以注入 Mock
mockUC := &MockCreateUseCase{
    result: &game_mgmt.CreateResult{Game: &entity.Game{ID: "test-id"}},
}
handler := NewGameHandler(mockUC, nil)
// 現在可以測試 Handler 的邏輯，不需要真的資料庫
```
