# 01 — 專案總覽

## 這個專案在做什麼？

KFC Project 是一個 **線上遊戲平台的後端系統**，負責管理遊戲設定、處理玩家遊戲邏輯、以及提供管理後台。它不是一個單體應用，而是由多個 **微服務 (Microservices)** 組成的分散式系統。

## 各服務職責

### 1. admin-backend（管理後台）
- **位置**：`/admin-backend/server/`
- **技術**：Go + Gin（HTTP）、MongoDB、Redis、JWT
- **職責**：
  - 管理員登入與認證（JWT Token）
  - 遊戲管理（新增、修改、上下架）
  - 整合商 (Integrator) 管理
  - 幣別管理與匯率設定
  - 權限群組（RBAC 角色控管）
  - 維護公告管理
  - 審計日誌
- **前端**：`/admin-backend/web/`，使用 Vue 3 + Vben Admin

### 2. config-service（設定同步中心）
- **位置**：`/config-service/`
- **技術**：Go + Gin、MongoDB Change Streams、SSE
- **職責**：
  - 提供遊戲、整合商、幣別等設定資料的查詢 API
  - 監聽 MongoDB 變更，透過 **SSE（Server-Sent Events）** 即時推播給其他服務
  - 是 admin-backend 與 gameservice 之間的橋樑

### 3. sfc-stream-game（遊戲引擎）
- **位置**：`/sfc-stream-game/`
- **包含三個子服務**：
  - **gameservice**：核心遊戲邏輯（gRPC），處理玩家下注、開獎
  - **gameapi**：HTTP API 包裝層，對外提供 RESTful 介面
  - **connector**：WebSocket 連線管理，處理玩家即時通訊

### 4. 支援模組
| 模組 | 位置 | 用途 |
|------|------|------|
| **math-lib** | `/math-lib/` | 共用的遊戲數學運算函式庫 |
| **go-observability** | `/go-observability/` | OpenTelemetry 可觀測性工具庫 |
| **kfc-k8s** | `/kfc-k8s/` | Kubernetes 部署清單 (Kustomize) |
| **Special_Game_Pipeline** | `/Special_Game_Pipeline/` | 測試工具、Mock 服務、基礎設施 Docker Compose |

## 給 Junior 的重點提示

> **Q：為什麼不用一個大的程式就好？為什麼要拆成多個服務？**
>
> A：微服務架構讓每個服務可以 **獨立部署、獨立擴展、獨立開發**。當遊戲流量爆增時，只需要擴展 gameservice，不需要連 admin-backend 一起擴。每個服務也有自己的責任邊界，團隊成員可以專注在自己負責的服務上。
>
> 但微服務也帶來了複雜度——服務之間怎麼溝通？資料怎麼同步？出錯了怎麼補償？這些都是後面章節會深入探討的主題。
