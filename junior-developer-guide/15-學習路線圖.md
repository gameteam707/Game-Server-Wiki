## 第一階段：Go 語言基礎（1-2 個月）

### 必須掌握

| 主題 | 重要性 | 學習資源 |
|------|--------|---------|
| **Interface（介面）** | 極高 | 本專案的 Port 全部都是 interface |
| **Struct & Method** | 極高 | 理解 receiver、embedding |
| **Error Handling** | 極高 | Go 沒有 exception，用 error 回傳 |
| **Goroutine & Channel** | 高 | SSE Hub、背景 Worker 都用到 |
| **Context** | 高 | 每個函式都傳 context.Context |
| **Package 管理** | 高 | go mod、internal/ 的意義 |
| **Testing** | 高 | 能寫單元測試、Table-Driven Tests |
| **Struct Tags** | 中 | json、bson 標籤的用途 |

### 建議學習路徑

```
1. Go Tour（官方互動教學）
   → https://go.dev/tour/
   → 2-3 天完成

2. Effective Go
   → https://go.dev/doc/effective_go
   → 1 週閱讀，重點在 Interface 和 Concurrency

3. 動手做：在本專案中加一個小功能
   → 例如：新增一個「遊戲供應商」的 CRUD
   → 會讓你實際走過 Domain → UseCase → Adapter 完整流程

4. Go by Example
   → https://gobyexample.com/
   → 隨時查閱的參考資源
```

### 推薦書籍

- *The Go Programming Language*（Donovan & Kernighan）— Go 語言聖經
- *Learning Go*（Jon Bodner, O'Reilly）— 更現代、實戰導向
- *100 Go Mistakes and How to Avoid Them*（Teiva Harsanyi）— 避坑指南

## 第二階段：架構與設計模式（2-3 個月）

### 必須掌握

| 主題 | 重要性 | 在本專案的體現 |
|------|--------|--------------|
| **Clean Architecture** | 極高 | 整個專案的架構基礎 |
| **Dependency Injection** | 極高 | DI Container |
| **Repository Pattern** | 極高 | 資料存取層 |
| **SOLID 原則** | 高 | 特別是 SRP 和 DIP |
| **Interface Segregation** | 高 | Port 介面設計 |
| **Middleware Pattern** | 高 | HTTP 中介軟體鏈 |
| **DTO Pattern** | 中 | Request/Response 物件 |
| **Command Pattern** | 中 | UseCase 的 Command/Result |

### 建議學習路徑

```
1. 閱讀 Clean Architecture（書籍或文章）
   → 理解依賴規則、分層原則

2. 對照本專案程式碼
   → 每讀一個模式，就在程式碼中找到對應的實作
   → 用紙筆畫出依賴關係圖

3. 嘗試從零建立一個小的 Clean Architecture 專案
   → 只做一個功能（如 TODO List）
   → 但要完整走過四層

4. 閱讀 Domain-Driven Design 入門
   → Entity、Value Object、Aggregate 的概念
```

### 推薦書籍

- *Clean Architecture*（Robert C. Martin）— 架構設計的經典
- *Head First Design Patterns*（Freeman & Robson）— 設計模式入門
- *Domain-Driven Design Distilled*（Vaughn Vernon）— DDD 精簡版

## 第三階段：分散式系統基礎（2-3 個月）

### 必須掌握

| 主題 | 重要性 | 在本專案的體現 |
|------|--------|--------------|
| **HTTP/REST API 設計** | 極高 | admin-backend, gameapi |
| **gRPC 基礎** | 高 | gameservice |
| **MongoDB 操作** | 高 | 所有服務的資料存取 |
| **Redis 基礎** | 中 | 快取、限流 |
| **訊息佇列概念** | 中 | Kafka 事件驅動 |
| **SSE/WebSocket** | 中 | config-service, connector |
| **CAP 定理** | 中 | 理解分散式系統的取捨 |
| **Saga/Outbox Pattern** | 中 | gameservice 的交易補償 |

### 建議學習路徑

```
1. 從 HTTP API 開始
   → 用 Postman 呼叫本專案的 API
   → 理解 RESTful 慣例

2. 學習 MongoDB
   → 安裝 MongoDB Compass
   → 練習基本的 CRUD 操作
   → 理解 Document 模型 vs SQL

3. 接觸 gRPC
   → 閱讀本專案的 .proto 檔案
   → 嘗試用 grpcurl 呼叫 gameservice

4. 理解分散式系統挑戰
   → Martin Kleppmann 的演講
   → 重點：一致性、可用性、分區容錯
```

### 推薦書籍

- *Designing Data-Intensive Applications*（Martin Kleppmann）— 分散式系統聖經
- *Microservices Patterns*（Chris Richardson）— 微服務模式大全
- *Building Microservices*（Sam Newman）— 微服務入門

## 第四階段：DevOps 與營運（持續學習）

### 必須掌握

| 主題 | 重要性 | 在本專案的體現 |
|------|--------|--------------|
| **Docker** | 高 | 多階段建置、Docker Compose |
| **Kubernetes 基礎** | 中 | Pod、Deployment、Service |
| **CI/CD 概念** | 中 | 自動化建置與部署 |
| **可觀測性** | 中 | OTel、Prometheus、Grafana |
| **日誌管理** | 中 | 結構化日誌、Loki |

### 建議學習路徑

```
1. Docker
   → 學會寫 Dockerfile、docker-compose.yml
   → 在本地建置並運行本專案的服務

2. Kubernetes
   → 先用 minikube 或 Docker Desktop K8s
   → 嘗試部署一個簡單的服務
   → 再來理解本專案的 kfc-k8s 結構

3. 可觀測性
   → 啟動 Grafana，探索儀表板
   → 學會用 Prometheus 查詢指標
   → 理解 Trace 和 Span 的概念
```

## 每日練習建議

### 第 1-2 週：讀懂程式碼

```
每天花 30 分鐘：
□ 選一個功能（如「建立幣別」）
□ 從 Router → Handler → UseCase → Repository 完整追蹤
□ 畫出呼叫鏈和資料流
□ 記下不懂的地方，去查或問
```

### 第 3-4 週：動手修改

```
每天花 1 小時：
□ 嘗試加一個簡單的功能（如新增一個查詢 API）
□ 跑測試確認沒有弄壞東西
□ 寫一個新的測試案例
□ 用 Docker 建置看看
```

### 第 2 個月起：獨立開發

```
嘗試獨立完成：
□ 新增一個完整的 Entity + UseCase + Adapter
□ 從 .proto 定義到 gRPC handler 的完整流程
□ 為你寫的程式碼加上完整的測試
□ 使用 Prometheus 為你的功能加上指標
```

## 心態建議

1. **不要急**：這個專案的架構是經驗豐富的工程師設計的，不可能一週就完全理解。給自己 2-3 個月。

2. **先理解「為什麼」**：每看到一個設計決策，先問「為什麼這樣做」而不是急著記住「怎麼做」。

3. **善用 IDE**：用 GoLand 或 VS Code + Go 擴充，善用「Go to Definition」和「Find Usages」追蹤程式碼。

4. **畫圖**：用紙筆或工具畫出架構圖、呼叫鏈、資料流。視覺化是理解複雜系統的最佳方式。

5. **不恥下問**：不懂就問。每個 Senior 都是從不懂開始的。
